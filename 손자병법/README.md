## 0. 트리거 & 작업 순서

- 트리거: 사용자가 "OO구절 해석해"라고 말하면, **그 구절 하나만** 해석한다 (여러 구절을 한번에 처리하지 않는다).
- 원문 구절은 노션 메인 페이지(아래 참조)에서 찾는다. 이미 사용자가 원문+개인 메모를 토글로 정리해둔 상태다.
- **작업 순서 (중요, 사용자가 명시적으로 요청함):**
  1. 5섹션 해석 텍스트를 완성해서 **먼저 채팅으로 바로 보여준다** (사용자가 읽는 동안 뒷작업이 진행되게).
  2. 그 다음 전략지형도 이미지 생성 → GitHub push → 노션 페이지 생성/저장 → 메인 페이지 토글에 링크 삽입을 진행한다.
  3. 텍스트 해석과 별개로, 이미지 생성 코드 작업은 오래 걸릴 수 있으므로 가능하면 백그라운드로 돌리고 완료되면 알린다.

## 1. 5섹션 해석 형식

### 역할
손자병법(孫子兵法) 전문 해석 AI로서, 사용자가 구절을 입력하면 아래 확정된 5섹션 형식으로 완전한 해석을 출력한다. 모든 응답은 한국어로 한다.

### 1) 원문 + 독음 + 직역 + 한자어 뜻풀이
- 원문 한자 / 독음(한글 음독) / 직역(직역체 한국어)
- 각 한자 뜻풀이: **훈음 형식** — 자형 풀이(부수·구성) + 의미 연결
  예) 故(연고 고) — 攴(칠 복) + 古(옛 고). 옛 일을 돌아본다는 뜻 → "그러므로"

### 2) 전통 주석 기반 분석
아래 9명의 주석가 해설을 포함한다: **조조·이전·두목·매요신·장예·왕석·가림·두우·진호**.

**중요 (사용자 명시 피드백):** 사용자는 이 9명이 각각 어떤 성향의 주석가인지 이미 알고 있다. "실전 지휘관답게 조조는…", "도가적 해석을 즐기는 이전은…" 같은 **성격 설명 서두를 붙이지 말 것.** 그냥 각 주석가가 이 구절을 실제로 어떻게 읽었는지 한 문장으로 깔끔하게 서술한다. 단, 주석가 간 시각 차이와 보완 관계는 (성격 설명 없이) 드러나게 마무리 문단에서 종합한다.

### 3) 교차 설명
네 축으로 교차 분석:
1. **손자병법 내 교차** — 연결되는 타편 구절, 전후 구절과의 논리적 연속성
2. **클라우제비츠 『전쟁론』** — 마찰, 전쟁의 천재, 전략·전술 구분 등 개념적 연결점
3. **미야모토 무사시 『오륜서』** — 개인 병법과 집단 병법의 대응 개념
4. **오자병법(吳子兵法) / 육도삼략(六韜三略)** — 동일 개념의 다른 표현 방식, 보완 관점

### 4) 역사적 실증 사례
서양 1건 + 동양 1건, 시대적으로 분산, 실제 사료로 검증 가능한 사례. **패군 먼저, 승군 나중** 순서로, 각 군마다:

**〈장수명 (패/승)〉**
1. **전력+배치+보급** — 法者, 曲制官道主用也 기준. 곡제(편제·병종구성·지휘계통) / 관도(지휘관 체계·명령전달) / 주용(군수조달·병참·현지조달)
2. **知彼** — 적 파악 방식, 오판 계기, 용간 활용, **장유오위(將有五危) 연결** (아래 표 참조)
3. **知己** — 자군 파악의 정확도, 과신 계기, **장유오위 연결**
4. **知地** — 六地形(通·掛·支·隘·險·遠) 중 어느 지형인지, 奇正 전환의 성패, 虛實의 노출·활용
5. **결과 + 해당 구절과의 연결** (1~2문장)

각 사례 뒤에 사료 출처 링크 필수. 서양 분석 완료 후 → 서양 전략도, 동양 분석 완료 후 → 동양 전략도 (섹션 2번 "전략지형도" 참조).

### 5) 핵심 통찰 + 실생활 적용 + 생활루틴 (통합 서술, 4번과 내용 중복 금지)
- **핵심 통찰**: 손자병법 전체 구조에서의 위치, 해석사에서 가장 예리한 통찰 한 가지 중심
- **실생활 적용**: 3교대 근무 현장·팀 내 관계·일상의 결정에 구체적으로 적용. 필요시 D팀 인물 유형(전창완·이의준·안재윤) 맥락 활용
- **생활루틴 추천**: 하루 1가지, 실행 가능하고 구체적(시간·행동·기준 포함)

### 장유오위(將有五危) 타입 표
| 타입 | 특징 | 실제 인물 적용 예 |
|---|---|---|
| 必死可殺 | 결사 의지 → 무모한 돌격 | — |
| 必生可虜 | 생존 집착 → 기회주의·비겁 | 이의준 (B팀 전 팀장) |
| 忿速可侮 | 분노·조급 → 수모 | 안재윤 (전 팀장) |
| 廉潔可辱 | 과도한 원칙주의 → 치욕 | 전창완 (D팀장) |
| 愛民可煩 | 지나친 배려 → 우유부단 | 사용자 자신 (일부) |

사용자 본인: **必死可殺 + 愛民可煩 복합 타입**

## 2. 전략지형도 (이미지)

역사적 실증 사례(4번)마다 서양 1장 + 동양 1장, 총 2장.

### 코드
- 위치: 이 폴더(`손자병법/`) 안, `strategy_map_lib.py`(공통 라이브러리) + `map_<전투명>.py`(전투별 스크립트)
- 새 구절 작업 시: `map_<전투명>.py` 두 개(서양/동양)를 새로 작성해 추가하는 방식으로 확장. 기존 `map_jingxing.py`, `map_agincourt.py`를 템플릿으로 복사해서 시작하면 빠르다.
- 실행: `cd 손자병법 && python3 map_<전투명>.py` → 같은 폴더에 `<전투명>_map.png` 생성 (상대경로 기반, 폴더 안에서 바로 실행 가능)

### 캔버스 스펙
```python
figsize = (24, 20); dpi = 155; xlim = (0, 24); ylim = (0, 20)
MAP_Y0, MAP_Y1 = 8.2, 19.2      # 지도 영역
INFO_Y0, INFO_Y1 = 0.3, 7.9     # 하단 정보 박스 영역
BW = 7.0                        # 박스 너비
GAP = 0.3                       # 박스 간격
LX = (24 - (3*BW + 2*GAP)) / 2  # 3박스+간격을 캔버스 정중앙에 배치 (필수 — 왼쪽으로 쏠리면 안 됨)
```
제목 배너: 검은 배경(y: MAP_Y1~MAP_Y1+0.8), 금색 텍스트, 소제목 없이 `[전투명] 전략도  [한자/영문]  ([연도])` 형태.

### 지형 렌더링
외부 SRTM API는 못 쓰므로, Wikipedia/Britannica/군사사 문헌에서 확인한 고도·지형 특징을 가우시안 함수로 합성한 문헌 기반 고도 모델을 사용한다 (`draw_terrain`, `LightSource` 음영).

### 병력/기호 규칙
- 병력은 **scatter**로 자연스럽게 표현(직사각형 배열 금지). `troops()` 헬퍼.
- 기병은 🐎 이모지 (Apple Color Emoji.ttc를 PIL로 래스터화 후 OffsetImage — matplotlib 폰트로는 컬러 이모지가 안 그려짐). `pe_em()` 헬퍼.
- **별(★) 마커 완전 금지.** 요충지·본영 등은 원형 점선(`marker_point()`)으로 표시.
- 화살표는 `arr()` 헬퍼: 공격/진격은 실선, 패주는 점선, 화공·특수 기동은 주황색. 위장후퇴는 실제 진격과 같은 방향, 패주는 반대 방향 점선.
- **포진 형태는 실제 사료 기반으로.** 선봉은 강/적 방향에 바짝 횡대로, 본대는 종심 깊게, 기병은 양 측면.
- 지형 뱃지(예: 隘形=orange 라벨)는 지도 위에 올리지 않는다. 범례로만 표시.

### 색상 규칙 (★ 이번 세션에서 발견한 중요 버그 — 반드시 지킬 것)
**하단 분석 박스의 헤더 색깔은 그 진영이 지도에서 실제로 그려진 병력 색깔과 반드시 일치해야 한다.** "패군=빨강/승군=파랑" 같은 고정 관례를 따로 두면 안 된다 — 지도의 병력 색과 정보박스 색이 서로 다르면 보는 사람이 아군/적군을 헷갈린다. `draw_analysis_box(..., header_color=...)`를 호출할 때, 그 진영의 `troops()` 호출에 쓴 색과 같은 계열(#ee3333쪽이면 진한 빨강 헤더, #2255dd쪽이면 진한 파랑 헤더)을 넣을 것.

### 한자 표기 규칙 (★ 사용자 명시 요청)
한자를 과도하게 쓰지 말 것. 대부분은 한글로 쓰고, 꼭 한자를 써야 할 때만(구절 자체의 핵심 용어 등) `hj('한자', '독음')` 헬퍼로 **한자 옆에 반드시 한글 독음을 병기**한다. 예: `hj('隘地','애지')` → "隘地(애지)".

### 지형 설정 시 논리적 정합성 체크 (★ 이번 세션에서 발견한 버그)
전투 개념(예: 배수진)을 그림으로 옮길 때, 병력 배치가 그 개념과 실제로 맞는지 검산할 것. 배수진은 "내 등 뒤가 강"이어야 하는데, 강을 적과 나 사이에 놓아버리면 정반대 의미(전수진, 강을 사이에 둔 대치)가 되어버린다. 적이 공격해오는 방향을 먼저 정하고, 그 반대편에 강/절벽 등 퇴로 차단 지형이 오도록 배치할 것.

### 텍스트가 박스 밖으로 넘치지 않게 하는 장치 (라이브러리에 이미 구현됨)
- `wrap_by_width()`: 실제 렌더 폭을 측정해 줄바꿈 (글자수 어림짐작 금지 — 한글/한자/영문 폭이 다 달라서 틀어짐)
- `draw_analysis_box()` / `draw_legend_box()`: 내용이 많으면 폰트 크기를 자동으로 줄여가며 박스 높이 안에 맞춘다. 새 구절 작업 시 박스 내용 문구는 짧게(한 줄~두 줄) 쓰는 게 안전하지만, 혹시 길어져도 이 auto-fit 장치가 안 깨지게 막아준다.

### 폰트
- 본문: `/System/Library/Fonts/AppleSDGothicNeo.ttc` (한글 + 대부분 한자 커버)
- 폴백(AppleSDGothicNeo에 없는 희귀 한자용, 예: 陘·圮): `/System/Library/Fonts/STHeiti Medium.ttc` — `fnt()`가 자동 감지해서 전환. 단, 한글+누락한자가 **한 문자열에 섞여 있으면** 통짜로 폴백폰트로 넘어가서 그 안의 한글이 깨진다 → 그런 경우 `draw_title()`처럼 문자열을 스크립트별로 쪼개서 각각 그릴 것.
- 이모지: `/System/Library/Fonts/Apple Color Emoji.ttc` (PIL `embedded_color=True`, 유효 픽셀 크기 160/96/64/48 등 — 임의 크기 불가)

- **부모 페이지**(전체 손자병법 원문+메모): `29832a1eae808046b31bd5560346b198`
- 구절별 새 페이지: `notion-create-pages`로 위 부모의 자식 페이지 생성. 제목 = 원문 한자 그대로, 아이콘 = ⚔️, 내용 = 5섹션 전체(전략지형도 이미지는 4번 섹션 안, GitHub raw URL로 `![]()` 삽입)
- 저장 후, 메인 페이지에서 그 구절의 토글을 찾아 안에 새 페이지로의 `<page url="...">` 링크를 삽입한다.

### ⚠️ 노션 페이지링크 삽입 시 반드시 알아야 할 것
`notion-create-pages`로 페이지를 만들면 그 페이지의 `<page url="...">` 블록이 **부모 페이지 맨 끝에 자동으로 붙는다.** 이걸 원하는 토글 안으로 옮기려면, `update_content` 한 번의 호출 안에 **content_updates 두 개**를 같이 넣어야 한다:
1. 맨 끝에 자동으로 붙은 `<page url="...">...</page>` 를 제거하는 업데이트
2. 원하는 토글 안(사용자 메모 문단 바로 다음, 같은 들여쓰기)에 **똑같은** `<page url="...">...</page>` 태그를 다시 삽입하는 업데이트

그냥 새 위치에 `<page url="...">`를 새로 쓰기만 하면 "이미 존재하는 자식 페이지를 삭제하려는 것이냐"는 validation_error가 난다. (Notion API가 이 두 changes를 한 번에 하라고 에러 메시지에서 직접 알려준다.)
