# 북마크관리

크롬 북마크 일괄 점검/수정 도구. 새 세션에서도 이 문서만 보고 바로 이어서 쓸 수 있도록 정리.

- `fix_bookmarks.py`: 북마크 폴더(또는 전체) 안 URL을 점검해서 정상/리다이렉트/깨짐을 분류하고, 리다이렉트는 최종 주소로, 깨진 링크는 Wayback Machine 저장본으로(`--archive`) 자동 교체 후보를 보여준다. `--replace OLD NEW`로 문자열 단순 치환도 가능(접속 확인 없이). 항상 `--apply`를 붙여야 실제로 파일에 반영되고(기본은 미리보기만), 반영 전 원본은 자동으로 `Bookmarks.bak`로 백업된다.
- `run_fix_bookmarks.sh`: `fix_bookmarks.py`를 감싸는 래퍼. **크롬이 켜진 채로 `--apply`하면 크롬이 종료될 때 방금 고친 내용을 메모리에 있던 옛날 값으로 덮어써버리는 문제**(2026-07-22 확인)가 있어서, `--apply`가 인자에 있고 크롬이 실행 중이면 자동으로 크롬을 종료 → 스크립트 실행 → 크롬 재실행까지 처리한다. `--apply` 없이 미리보기만 할 때는 크롬을 건드리지 않는다.

## 사용법

```bash
# 특정 폴더(예: "天") 안의 북마크만 대상 — 폴더 이름을 반드시 --all 없이 지정
./run_fix_bookmarks.sh "폴더이름" --replace "OLD" "NEW" --apply

# 폴더 구분 없이 전체 북마크 대상 (★ 폴더를 함께 주면 --all이 그 폴더 지정을 무시하고 전체를 대상으로 하니 주의)
./run_fix_bookmarks.sh --all --replace "OLD" "NEW" --apply

# 링크 상태 점검만 (미리보기, 크롬 안 건드림)
./run_fix_bookmarks.sh "폴더이름" --archive
```

- 기본 대상 파일: `~/Library/Application Support/Google/Chrome/Default/Bookmarks` (다른 프로필은 `--bookmarks-file`로 지정).
- 의존성: `requests` (anaconda python3에 이미 설치돼 있음).

## 알아둘 것

- **`--all`과 폴더 이름을 같이 주면 폴더 이름은 무시되고 전체 북마크가 대상이 된다.** 특정 폴더로 범위를 제한하고 싶으면 `--all`을 빼고 폴더 이름만 줄 것 (2026-07-23, 실수로 `--all`을 같이 써서 의도보다 넓은 범위가 대상이 됐던 적 있음 — 결과적으로 피해는 없었지만 헷갈리기 쉬움).
- `--replace`는 접속 확인 없이 문자열만 바꾸는 거라 서버 이전(예: `kr42` → `kr43` 같은 서브도메인 변경)처럼 URL 패턴이 뻔히 바뀐 걸 알 때 빠르게 쓰기 좋다. 실제로 살아있는지/리다이렉트되는지 확인하려면 `--replace` 없이 `--archive`로 점검부터 할 것.
